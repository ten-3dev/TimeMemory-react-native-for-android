왜 필요한가?
	한마디로 말하자면 과거의 몇시에 내가 무엇을 했는지 기록 하기 위함이다.
	살다보면 내가 언제인지는 상관이 없지만 몇시에 무얼 시작했는지 끝냈는지 궁금할때가 있다. 그걸 기록하기 위함이다.

기능?
	1. 현재 날짜를 기준으로 간단한 텍스트와 시간이 들어간다.
	2. 간단한 CRUD를 구현할 것이며, 디테일적인 부분은 다음과 같다.
		2-1. 생성하는 곳으로 가면 현재 날짜와 시간이 defult다. 바꿀 수 없으며 간단한 텍스트만 허용한다.
		2-2. 다음은 업데이트 기능이다. 업데이트는 이 서비스에선 없다.
		2-3. 다음은 삭제 기능이다. 삭제는 View에서 할 수 있으며 또 다른 기능은 딱히 없다.
		2-4. 다음은 VIEW 기능이다. 가장 가까운 순으로 불러오며 삭제만 할 수 있다.
	3. 이 서비스에선 전체삭제 기능이 있다.
	4. 이 서비스에선 검색기능 또한 없기 때문에 기록에 limit이 있다.
	5. 이 서브스에선 다크모드를 지원하다. 이 기능은 AsyncStorage를 사용할 것이다. 

Design?
	디자인은 최대한 빈티지하며 간편하게 할 수 있도록 한다. 모든 기능 및 UI/UX는 최대한 빠르게 할 수 있어야 하며 쉽게 알아볼 수 있어야 한다.

Server API URL?
	*서버 API URL은 최대한 간결하게 한다.

	/get : View의 기능을 담당한다. 모든 기록을 불러오며 최신순으로 기록된다.

	/del?all=false : 이 기능은 삭제를 담당한다. 기록을 삭제 할 수 있다.

	/del?all=true : 이 기능은 삭제를 담당한다. 기록을 삭제 할 수 있다. all 파라미터의 default는 false이다.

*부가설명
	이 서비스는 웹이 아닌 앱이 관계로 front쪽 서버는 필요하지 않다.
	그러므로 back쪽 서버가 필요하다. 클라우드 서비스인 firebase를 사용할 것이다. 이 앱은 로그인 기능이 없다. 모두 내가 사용하는 것이며 다른 누군가가 사용하지 못한다. 최대한 원터치, 원스크롤로 기능을 해야하며 모든게 빠르게 이루어져야한다.
